# OHOS SDL2 上树流程说明

> 「上树」：把 SDL 的 XComponent Surface 挂进 ArkUI 节点树，使 SDL 能在屏幕上渲染并接收输入。

---

## 一、完整流程（7 步）

### 第 1 步：ArkUI 放 XComponent（`Index.ets`）

```typescript
XComponent({
  id: 'RootXC',
  type: XComponentType.NODE,
  libraryname: 'SDL2'   // 加载 libSDL2.so，触发 OHOS_XcomponentExport 注册回调
})
```

- ArkUI 渲染此组件时，系统自动调用 `OHOS_XcomponentExport`。
- `OHOS_XcomponentExport` 中通过 `OhosPluginManager::SetNativeXComponent` 把 `OH_NativeXComponent`
  指针存入 `nativeXComponentMap["RootXC"]`，并注册 Key/Mouse/Touch/Focus 等回调。
- 此时 **仅有 ArkUI 侧的 NODE 壳**，还没有 native Surface。

---

### 第 2 步：`onAppear` 触发，创建根节点

```typescript
// Index.ets → .onAppear(() => {
sdl.setScaledDensity(scaledDensity)
rootNode = sdl.createNativeNode('RootXC', this.mWidth, this.mHeight, this.x, this.y)
notifySdlAboutToAppear(getContext(this), rootNode)
```

- `sdl.createNativeNode` → C++ 侧 `OHOS_NAPI_CreateNativeNode` → `OHOS_AddRootNode`
- 在 C++ 里为 `'RootXC'` 创建 native 节点并写入 `m_map["RootXC"]`。
- 得到的 `rootNode` 指针传入下一步。

---

### 第 3 步：`OHOS_AddRootNode` —— 根节点真正上树

```cpp
// SDL_ohos.cpp → OHOS_AddRootNode(xcompentId, windowPosition)

adapter->GetNativeXComponent("RootXC")          // 从 EWAdapterC 取出已注册的 XComponent
XComponentModel *model =
    new XComponentModel("RootXC", SURFACE)      // ★ 只是参数描述结构体（id + type），
                                                //   本身不创建任何 ArkUI 节点
adapter->CreateRootNode(params, true)           // → 内部调用 CreateEWNode
```

**`CreateEWNode`（`SDL_AdapterC.cpp`）才是真正建节点的地方：**

```cpp
// SDL_AdapterC.cpp → CreateEWNode(nullptr, nodeParams, scale)

ArkUI_NodeHandle stack = nodeAPI->createNode(ARKUI_NODE_STACK);      // 建容器
ArkUI_NodeHandle xc    = nodeAPI->createNode(ARKUI_NODE_XCOMPONENT); // ← 第 ② 个 XC 在这里建
SetXCompontId(xc, "RootXC");
SetXCompontType(xc, ARKUI_XCOMPONENT_TYPE_SURFACE);
// 组装 Node { xc, stack } 并返回
```

```cpp
// 回到 OHOS_AddRootNode
m_map["RootXC"] = node                          // 写入节点管理表

OH_NativeXComponent *native =
    OH_NativeXComponent_GetNativeXComponent(adapter->GetXComponent(node));
OHOS_SetNativeXcomponent("RootXC", native);     // 重新注册回调

OH_NativeXComponent_AttachNativeRootNode(component, adapter->GetStack(node));
// ↑ 真正「上树」：把 native stack 附加到第 ① 个（ArkUI 壳）上，接入渲染管线
```

---

### 第 4 步：`notifySdlAboutToAppear` —— 启动 SDL 引擎

```typescript
// SdlModule.ets → notifySdlAboutToAppear(context, rootNode)

setResourceManager(context)           // 资源路径给 C++ 端
setScreenResolution()                 // 分辨率写入 C++ 全局变量
sdl.init(callbackRef)                 // 初始化 NAPI 回调（鼠标样式、方向等）
sdl.sdlRootNode(rootNode)             // 把 rootNode 指针送入 C++，供后续子窗口使用
sdl.sdlAppEntry("libentry.so", "main") // 在新线程里启动 SDL main()
```

- `sdlRootNode` → C++ `rootNode` 全局指针赋值（`OHOS_GetRootNode` 后续使用）。
- `sdlAppEntry` 开启新线程，执行 `libentry.so` 中的 `main()`，SDL 正式初始化。

---

### 第 5 步：SDL `CreateWindow` —— 子节点上树

```c
// SDL_ohoswindow.c → OHOS_CreateWindow

OHOS_GetRootNode(&parentWindowNode)                    // 拿到第 4 步存好的 rootNode
window->xcompentId = OHOS_GenerateNewXComponentId()   // 生成新 id："SDL2_XComponent_1"

OHOS_NAPI_AddChildNode(parentWindowNode,               // 把 SDL 窗口作为子节点挂到 root 下
                       &window->ohosHandle,
                       window->xcompentId,
                       &windowPosition)
// → OHOS_AddChildNode(SDL_ohos.cpp)
//     → adapter->AddChildNode(parentNode, params)（SDL_AdapterC.cpp）
```

**同样进入 `CreateEWNode`，第 ③ 个 XC 在此创建：**

```cpp
// SDL_AdapterC.cpp → CreateEWNode(parentNode, nodeParams, scale)

ArkUI_NodeHandle stack = nodeAPI->createNode(ARKUI_NODE_STACK);
ArkUI_NodeHandle xc    = nodeAPI->createNode(ARKUI_NODE_XCOMPONENT); // ← 第 ③ 个 XC 在这里建
SetXCompontId(xc, "SDL2_XComponent_1");
SetXCompontType(xc, ARKUI_XCOMPONENT_TYPE_SURFACE);
// 挂到 parentNode（根节点 stack）下面
```

```cpp
// 回到 OHOS_AddChildNode
m_map["SDL2_XComponent_1"] = node
OHOS_SetNativeXcomponent("SDL2_XComponent_1", native);  // 注册 Key/Mouse/Touch 回调
```

- 每次 `SDL_CreateWindow` 生成一个新 id，`CreateEWNode` 为其建一个独立 XC Surface。
- 多窗口场景下每个 SDL_Window 各自独立挂节点。

---

### 第 6 步：`OnSurfaceCreatedCB` —— Surface 就绪，写入 WindowData

```cpp
// SDL_ohos_xcomponent.cpp → OnSurfaceCreatedCB(component, window)

OH_NativeXComponent_GetXComponentSize(...)     // 获取 Surface 宽高
OH_NativeXComponent_GetXComponentOffset(...)   // 获取偏移

SDL_WindowData *data = SDL_malloc(sizeof(SDL_WindowData));
data->native_window = (OHNativeWindow *)window;
data->width  = width;
data->height = height;

OhosPluginManager::GetInstance()
    ->SetNativeXComponentList(component, data);
// ↑ 写入 nativeXComponentList[component] = data
//   让 OHOS_WaitGetNativeWindow 能轮询拿到
```

---

### 第 7 步：`OHOS_CreateWindowFrom` —— SDL_Window 完整建立

```c
// SDL_ohoswindow.c → OHOS_CreateWindowFrom

OHOS_FindNativeXcomPoment(strID, &nativeXComponent) // nativeXComponentMap[id] → XComponent

// 轮询等待（最多 700 × 10ms = 7s）
OHOS_WaitGetNativeXcompent(id, tid, &nativeXComponent)  // 等 XComponent 注册完毕
OHOS_WaitGetNativeWindow(id, tid, &windowData, ...)     // 等 Surface 回调写入数据

// 取到 native_window 后建 EGL Surface
sdlWindowData->native_window = windowData->native_window;
sdlWindowData->egl_xcomponent =
    SDL_EGL_CreateSurface(thisDevice, (NativeWindowType)windowData->native_window);

window->driverdata = sdlWindowData;  // SDL_Window 完整建立完毕
```

---

### 下树（`aboutToDisappear`）

```typescript
// Index.ets
sdl.removeNode('RootXC')           // 把根节点从树上摘掉 → m_map 清空
notifySdlAboutToDisappear()        // → nativeSendQuit() → SDL 退出
```

## 二、节点树结构与三个 XComponent 的对应

```
ArkUI Inspector 里看到的树
│
├── ① XComponent(RootXC) [type=NODE]          ← Index.ets ETS 声明，ArkUI 框架自动创建
│       libraryname='SDL2'，是「插槽/容器壳」
│       触发 OHOS_XcomponentExport 注册回调（第 1 步）
│
│   ↓ OH_NativeXComponent_AttachNativeRootNode（第 3 步末尾）把 ② 挂进 ①
│
├── ② XComponent(RootXC) [type=SURFACE]       ← CreateEWNode 里
│       nodeAPI->createNode(ARKUI_NODE_XCOMPONENT)
│       由 adapter->CreateRootNode → CreateEWNode 创建（第 3 步）
│       是真正的 native 根 Surface，响应 OnSurfaceCreatedCB
│
│   ↓ adapter->AddChildNode → CreateEWNode（第 5 步）
│
└── ③ XComponent(SDL2_XComponent_1) [type=SURFACE]  ← 同一个 CreateEWNode
        nodeAPI->createNode(ARKUI_NODE_XCOMPONENT)
        SDL_CreateWindow 时由 OHOS_AddChildNode 触发（第 5 步）
        SDL 游戏内容渲染在此，EGLSurface 绑定到此 native_window（第 6~7 步）
```

> `XComponentModel` 是**参数描述结构体**（只带 id + type），不创建节点。  
> 真正创建 ArkUI 节点的 API 是 `SDL_AdapterC.cpp` 里的  
> `nodeAPI->createNode(ARKUI_NODE_XCOMPONENT)`。

## 三、四张哈希表分工

| 表名 | Key | Value | 写入时机 | 用途 |
|------|-----|-------|----------|------|
| `m_map` | `xcompentId` (string) | `shared_ptr<Node>` (ArkUI Node) | `OHOS_AddRootNode` / `OHOS_AddChildNode` | Show/Hide/Move/Resize/Remove 节点 |
| `nativeXComponentMap` | `xcompentId` (string) | `OH_NativeXComponent *` | `OHOS_XcomponentExport` / `OHOS_SetNativeXcomponent` | id → NDK 对象，供查找和注册回调 |
| `nativeXComponentList` | `OH_NativeXComponent *` | `SDL_WindowData *` | `OnSurfaceCreatedCB` | XComponent → SDL 窗口数据，供事件派送 |
| `threadXcompentList` | `pthread_t` | `vector<string>` (id 列表) | `OHOS_AddXcomPomentIdForThread` | 线程归属，多线程清理用 |

## 四、新版与老版本（`sdlmodify`）主要差异

| 对比点 | 老版本（`sdlmodify`） | 新版本 |
|--------|----------------------|--------|
| XComponent `libraryname` | `nativerender` | `SDL2` |
| NAPI 入口模块 | `import qt from 'libentry.so'` | `import sdl from 'libSDL2.so'` |
| `sdlAppEntry` 参数 | `"libentry.so", "main", "testyuv.bmp"`（测试 demo） | `"libentry.so", "main"` |
| `OHOS_SetRealWindowPosition` | 无 NULL 检查（可能崩溃） | 有 NULL 检查 + 错误日志 |
| 调试 log | 大量 `SDL_Log("2222...")` 残留 | 清理后保留关键日志 |
| `OHOS_InitKeyboard` | VideoInit 里**未调用** | VideoInit 里已补上 |
| Num Lock 键映射 | 缺失（按 Num Lock 可能 UNKNOWN） | 已补 `KEYCODE_NUM_LOCK → SDL_SCANCODE_NUMLOCKCLEAR` |
| `syncKeyboardLockModifiers` NAPI | **无** | 已新增，供 ArkTS 同步 Caps/Num 状态 |
| `onNativeFocusChanged` | `(focus = SDL_TRUE ? ...)` 永远 GAINED | 已修正为正确条件分支 |
